# Phase 158 - Fix: Pre-check Database Connection in Cache Refresh

## Vấn đề
- Khi chạy `DEPLOY-SERVER.sh`, lệnh `dashboard:refresh-cache` thất bại với 60+ lỗi "Access denied" giống nhau
- Mỗi cache key đều thử query database riêng biệt → hiển thị 60+ dòng lỗi giống nhau
- Output deploy rất khó đọc, khó phân biệt lỗi thật

## Giải pháp
- Thêm pre-check kết nối MySQL tại đầu lệnh `dashboard:refresh-cache`
- Nếu kết nối thất bại → hiển thị **1 thông báo lỗi rõ ràng** + gợi ý nguyên nhân, rồi thoát sớm
- Không cố chạy 60+ queries sẽ fail

## File thay đổi
- `src/app/Console/Commands/RefreshDashboardCache.php`: Thêm database connection pre-check trong `handle()`
